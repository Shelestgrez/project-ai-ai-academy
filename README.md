# AI Academy Web App

`AI Academy` — учебное веб-приложение по основам искусственного интеллекта.  
Проект создан как простой и понятный старт для пользователей, которые хотят изучать ИИ по коротким урокам с практикой и мини-тестами, а также видеть свой прогресс в личном кабинете.

## Зачем сделан проект

Основная цель проекта — дать базовое, структурное и доступное обучение ИИ без сложной установки и без технического порога.

Проект решает сразу несколько задач:

- знакомит с ключевыми понятиями ИИ, ML и нейросетей;
- даёт интерактивный формат обучения (урок + quiz + отметка выполнения);
- хранит прогресс каждого пользователя;
- поддерживает мультиязычность (русский, английский, казахский);
- запускается в один клик на Windows.

## На чём сделан проект

Технологический стек:

- `Python 3.11+` — основной язык разработки;
- `Flask` — backend-фреймворк и маршрутизация;
- `SQLite` (`database.db`) — хранение пользователей и прогресса;
- `Jinja2` — серверный рендеринг HTML-шаблонов;
- `HTML/CSS + Bootstrap` — интерфейс приложения;
- `Werkzeug security` — хеширование и проверка паролей.

## Ключевые функции

### 1) Пользователи и авторизация

- регистрация нового пользователя;
- вход в аккаунт;
- выход из аккаунта;
- хранение паролей в виде хеша (не в открытом виде).

### 2) Учебный контент

- интерактивные уроки по темам ИИ;
- блоки урока: обзор, шаги, разделы, практическое задание, глоссарий;
- тест после урока с автоматической проверкой.

### 3) Личный кабинет и прогресс

- список уроков с текущим статусом;
- процент общего прогресса по доступным урокам;
- сохранение результата quiz (баллы и количество вопросов);
- ручная отметка урока как пройденного.

### 4) Мультиязычность

Поддерживаются 3 языка:

- русский (`ru`);
- английский (`en`);
- казахский (`kk`).

Язык интерфейса и контента выбирается через параметр `lang` и сохраняется в сессии пользователя.

### 5) Готовность к развёртыванию на другом ПК

- автозапуск через `start_ai_academy.bat`;
- автоматическое создание `.venv` и установка зависимостей;
- отдельный скрипт для создания ярлыка на рабочем столе.

## Архитектура (кратко)

- основной файл приложения: `app.py`;
- шаблоны страниц: папка `templates`;
- статические файлы (стили, изображения): папка `static`;
- база данных: `database.db`;
- зависимости: `requirements.txt`;
- Windows-скрипты запуска/ярлыка:
  - `start_ai_academy.bat`
  - `create_desktop_shortcut.ps1`

## Основные страницы и маршруты

- `/` — главная страница;
- `/register` — регистрация;
- `/login` — вход;
- `/logout` — выход;
- `/dashboard` — кабинет и прогресс (только для авторизованных);
- `/lesson/<lesson_id>` — страница урока;
- `/quiz/<lesson_id>` — отправка теста (POST);
- `/complete/<lesson_id>` — ручная отметка завершения урока (POST).

## Структура базы данных

### Таблица `users`

- `id` — идентификатор пользователя;
- `name` — имя;
- `email` — уникальный email;
- `password_hash` — хеш пароля;
- `created_at` — дата создания.

### Таблица `progress`

- `id` — идентификатор записи;
- `user_id` — ссылка на пользователя;
- `lesson_id` — идентификатор урока;
- `completed` — статус завершения (0/1);
- `quiz_score` — набранные баллы;
- `total_questions` — количество вопросов;
- `updated_at` — дата обновления.

## Локальный запуск (Windows)

### 1) Требования

- установлен `Python 3.11+`;
- при установке Python включена опция `Add Python to PATH`.

### 2) Быстрый запуск в 1 клик

В корне проекта запустите:

`start_ai_academy.bat`

Скрипт автоматически:

1. создаст виртуальное окружение `.venv` (если отсутствует);
2. установит зависимости из `requirements.txt`;
3. запустит Flask-приложение.

После запуска откройте:

`http://127.0.0.1:5000`

## Перенос проекта на другой ПК

### Вариант A: перенос с сохранением пользователей и прогресса

1. Скопируйте **всю папку проекта** на другой ПК.
2. Убедитесь, что файл `database.db` также скопирован.
3. Запустите `start_ai_academy.bat`.

### Вариант B: чистый запуск без старых данных

1. Скопируйте папку проекта.
2. Не переносите `database.db` (или удалите его перед первым запуском).
3. Запустите `start_ai_academy.bat` — база создастся заново.

Примечание: папку `.venv` переносить не обязательно, она может быть пересоздана автоматически.

## Деплой в интернет (Render)

Проект подготовлен для деплоя на [Render](https://render.com/) через файл `render.yaml`.

### 1) Загрузите проект в GitHub

1. Создайте новый репозиторий на GitHub.
2. Загрузите туда файлы проекта (включая `render.yaml`).

### 2) Создайте сервис на Render

1. Войдите в Render и выберите **New +** -> **Blueprint**.
2. Подключите GitHub-репозиторий с этим проектом.
3. Render автоматически прочитает `render.yaml` и создаст web service.

### 3) Что настраивается автоматически

- установка зависимостей из `requirements.txt`;
- запуск через `gunicorn` (`app:app`);
- генерация `SECRET_KEY`.

### 4) После деплоя

- получите публичный URL вида `https://your-service-name.onrender.com`;
- откройте его в браузере и проверьте регистрацию/вход/уроки.

Примечание: на бесплатном тарифе Render возможны "засыпания" сервиса при простое и более долгий первый отклик.
Также в бесплатном варианте данные SQLite могут сбрасываться после пересоздания контейнера. Для постоянного хранения лучше подключить PostgreSQL.

## Создание ярлыка на рабочем столе

В папке проекта выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\create_desktop_shortcut.ps1
```

После этого появится ярлык `AI Academy` на рабочем столе.

## Остановка приложения

- закройте окно, где запущен `start_ai_academy.bat`;  
или
- нажмите `Ctrl + C` в активном окне терминала.

## Безопасность и ограничения текущей версии

- приложение ориентировано на локальное обучение/демо-использование;
- `SECRET_KEY` по умолчанию задан как dev-значение и для продакшена должен быть заменён через переменную окружения;
- используется SQLite, что отлично для локальной работы, но для высокой нагрузки лучше перейти на PostgreSQL/MySQL;
- отсутствуют отдельные автоматические тесты (pytest), рекомендуется добавить их на следующих этапах.
